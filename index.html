    Helps generate high quality CHANGELOG.md entries without constant conflicts.

    Project maintained by masukomi Hosted on GitHub Pages — Theme by mattgraham

    Change logs are an invaluable, and often neglected part of any software project.

    A good changelog helps you users to understand:

    • Why they should care about your latest version
    • If any of your changes affect the problems or frustrations they've been having.
    • If there are any changes that might affect how they use your app / library.
    • Why your efforts are worth their continued support.

    changelog_manager is a tool that makes it easy to generate high quality Changelog files that your users will want to read. Not auto-generated ðŸ’© from git commits.

    It encourages developers to create changelog entries at the correct level of abstraction, and it avoids the constant conflicts that would be generated my manually committing changes to CHANGELOG.md.
